v2.0 迁移指南

从 v1.x 平滑迁移到 v2.0，了解重大变更和新特性

📦 v2.3.0 · 最后更新：2026-01-27

概述

MCP Probe Kit v2.0 是一个重大版本更新，精简优化到 42 个核心工具，带来了更强大的功能和更好的开发体验。

💡

迁移时间：大多数项目可以在 10-15 分钟内完成迁移，v1.x 的所有工具在 v2.0 中都保持兼容。

主要变更

1. 工具精简优化

v1.x: 13 个工具 → v2.0: 42 个核心工具（移除低频工具，保留高价值工具）

2. 新增工作流编排

10 个 start_* 工作流编排工具，提供端到端的自动化开发流程：

  • start_feature - 功能开发工作流
  • start_bugfix - Bug 修复工作流
  • start_ui - UI 开发工作流
  • start_ralph - Ralph Wiggum Loop 自动化开发

3. 新增 UI/UX 工具集

完整的 UI/UX 工具链：

  • ui_design_system - 智能设计系统生成器
  • ui_search - UI/UX 数据库搜索
  • render_ui - UI 渲染引擎
  • design2code - 设计稿转代码

4. 100% 结构化输出

所有 42 个工具都支持结构化输出，确保输出格式一致、可预测。

迁移步骤

步骤 1：更新版本

Bash 
# 如果使用 npx（推荐）
# 无需操作，自动使用最新版本

# 如果全局安装
npm update -g mcp-probe-kit

步骤 2：检查配置文件

v2.0 的配置文件格式与 v1.x 完全兼容，无需修改。

✅

向后兼容：v1.x 的所有工具在 v2.0 中都保持兼容，可以无缝升级。

获取帮助

如果在迁移过程中遇到问题：

  • 📖 查看 完整工具参考
  • 💬 在 GitHub Discussions 提问
  • 🐛 在 GitHub Issues 报告 Bug
