安装配置

5 分钟快速上手 MCP Probe Kit v3.0.9，支持 Cursor、Cline 和 Claude Desktop

v3.0.9 MCP 2025-11-25 SDK 1.27.1
1

选择安装方式

无需安装，直接使用最新版本：

JSON 
{
  "mcpServers": {
    "mcp-probe-kit": {
      "command": "npx",
      "args": ["-y", "mcp-probe-kit@latest"]
    }
  }
}
优势：无需手动更新，每次启动自动使用最新版本

全局安装到系统：

Bash 
npm install -g mcp-probe-kit

配置文件：

JSON 
{
  "mcpServers": {
    "mcp-probe-kit": {
      "command": "mcp-probe-kit"
    }
  }
}
注意：需要手动运行 npm update -g mcp-probe-kit 更新版本

从源码编译安装，适合开发者：

1. 克隆仓库

Bash 
git clone https://github.com/mybolide/mcp-probe-kit.git
cd mcp-probe-kit

2. 安装依赖并编译

Bash 
npm install
npm run build

3. 配置文件

JSON 
{
  "mcpServers": {
    "mcp-probe-kit": {
      "command": "node",
      "args": ["/path/to/mcp-probe-kit/build/index.js"]
    }
  }
}
提示：Windows 用户使用完整路径，如 D:/workspace/mcp-probe-kit/build/index.js
注意：修改代码后需要重新运行 npm run build 编译
2

找到配置文件

根据你使用的 MCP 客户端，找到对应的配置文件：

在项目根目录创建或编辑 .cursor/mcp.json 或 .cline/mcp.json

💡 提示：Cursor 和 Cline 使用项目级配置，每个项目可以有独立的 MCP 服务器配置。

配置文件路径：

Path 
%APPDATA%\Claude\claude_desktop_config.json

示例：C:\Users\YourName\AppData\Roaming\Claude\claude_desktop_config.json

配置文件路径：

Path 
~/Library/Application Support/Claude/claude_desktop_config.json
3

验证安装

完成配置后，按以下步骤验证：

  1. 重启 MCP 客户端（Cursor / Cline / Claude Desktop）
  2. 在聊天窗口中输入：请使用 gencommit 工具生成一个测试提交消息
  3. 如果返回符合 Conventional Commits 规范的提交消息，说明安装成功！
✅ 成功示例：AI 返回类似 feat: 添加用户认证功能 的消息
❌ 常见问题：
  • 提示找不到工具 → 检查配置文件格式
  • 提示命令不存在 → 确认 Node.js 已安装
  • Windows 用户 → 确保使用 npx 方式
4

委托式编排协议

start_* 编排工具不会直接执行操作，而是返回一个可执行计划。

关键字段：mode: "delegated"、steps[]、outputs[]

Plan 示例 
{
  "mode": "delegated",
  "steps": [
    {
      "id": "spec",
      "tool": "add_feature",
      "args": { "feature_name": "user-auth" },
      "outputs": ["docs/specs/user-auth/requirements.md"]
    }
  ]
}
💡 要点：按顺序执行 steps，并确保 outputs 文件真实落盘

⚠️ 常见问题

错误信息： 
Error: Cannot find package 'yallist'

解决方案：清理 npx 缓存

Windows CMD 
rmdir /s /q %LOCALAPPDATA%\npm-cache\_npx
macOS/Linux 
rm -rf ~/.npm/_npx

然后重启 MCP 客户端

可能原因：
  • Node.js 版本过低（需要 >= 16.0.0）
  • 依赖未安装或损坏
  • 配置文件格式错误

解决方案：

  1. 检查 Node.js 版本： 
    node --version
  2. 如果低于 v16，请升级 Node.js
  3. 清理缓存并重新安装： 
    npm cache clean --force
  4. 检查配置文件 JSON 格式是否正确
